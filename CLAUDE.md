# FireHub — project context

## What this is
A static knowledge product for the FAO **Integrated Fire Management (IFM / MIF)**
work funded by Global Affairs Canada — project **GCP/GLO/1351/CAN**, internally
"FireHub". Coverage: Bolivia, Peru, Colombia, Ecuador and the Pan-Amazon (with
Brazil/IPAM educational materials). Documents are in Spanish, English and
Portuguese.

The current deliverable is a **browsable catalogue** of the source documents
(`index.html`), not yet a full synthesised knowledge site. It mirrors the
conventions of the sibling **DRIP** project at `C:\Dev\DRIP` (static HTML, a build
script that emits a `*-data.js` consumed by the page).

## Layout
- `index.html` — catalogue UI (search + Category/Type/Country facets + Clear all,
  category sections). Single inline script.
- `marvel.html` — MARVEL results page (Monitoring, Assessment, Reporting,
  Verification, Evaluation, Learning), by Marcelo Rezende / DSL-IP, applied to IFM.
  Framed as six **modular core functions** (assembled/customised per country context).
  Visualises the GCP/GLO/1351/CAN results-framework indicators + activity delivery;
  values are transcribed from the CRT spreadsheet into the inline `IND` / `groups`
  arrays. Every indicator/activity/gauge carries a **source tooltip** (native `title`)
  pointing to the CRT sheet/column. Uses `css/marvel.css`.
- `learnings.html` + `learnings.js` — the **Learning** page: lessons learned, best
  practices, key findings, technical recommendations distilled from the deliverables,
  clustered by theme and filterable by type/theme; each cites its source. Cards are
  summarised; clicking one opens a printable **A4 one-pager** (`#sheetOverlay`,
  deep-linkable via `learnings.html#FH-POL-01`, `window.print()` for the sheet only).
  Each item has a project code **`FH-<THEME>-<NN>`** (auto-generated by theme sequence)
  and fields `t` (title) / `x` (summary) / `why` (why it matters) / `s` (sources).
  Content is the `L` array (+ `SRC` map) in `learnings.js`; the script no-ops unless
  `#learngrid` exists. (DOCX sources were text-extracted to read them — see provenance.)
- `executive-summary.html` + `css/es.css` — **donor terminal executive summary** as
  print-ready A4 `.sheet` pages (cover · facts · results vs targets · 5 pillars/outputs ·
  photo essay · materials · countries · lessons · acknowledgements). Static HTML (no JS
  beyond a print button); `@page size:A4`, one page per `.sheet`, `.es-screenbar` hidden
  in print. Imagery in `assets/es-photos/` was mined from the project DOCX deliverables +
  the report's logos/charts (genuine project photos). Facts from the terminal report + CRT.
- `css/firehub.css` defines the shared `.topnav` linking Catalogue ↔ MARVEL ↔ Learnings ↔ Summary.
- `catalog-data.js` — **auto-generated**; never hand-edit.
- `scripts/build-catalog.mjs` — scans `GAC deliverables/`, writes `catalog-data.js`.
  Per file it prefers the frozen entry in `classification.json`, else falls back to
  the rules in `classify.mjs`.
- `scripts/classify.mjs` — shared, pure filename → category/country rules
  (`CATEGORY_RULES`, `COUNTRY_RULES`; first match wins).
- `scripts/classification.json` — **frozen** category/country per (renamed) file.
  Generated by `freeze-classification.mjs`; the build trusts it over the rules.
- `scripts/rename-map.mjs` — old → new filename map (single source of truth for names).
- `scripts/rename-files.mjs` — apply / `--check` / `--undo` renames; writes `rename-log.csv`.
- `scripts/freeze-classification.mjs` — runs each ORIGINAL name through the rules and
  stores the result against the NEW name (so renaming doesn't re-bucket files).
- `css/firehub.css` — catalogue layout ONLY; references the identity tokens via
  `var(--…)`. No hard-coded hex, no `@font-face` (Lato comes from the kit).
- `resources/fire-hub-identity/` — the **official FAO Global Fire Management Hub
  identity** (petrol/orange/rust, Lato, flat colour, accent-bar device). `index.html`
  links its `css/styles.css`; `tokens.json` + `css/tokens/*.css` are the source of
  truth. Its `AGENTS.md` lists non-negotiable brand rules — follow them.
- `GAC deliverables/` — the **source documents** (renamed to clean titles). Read-only input.

## Working rules
- After adding/removing files in `GAC deliverables/`, run `node scripts/build-catalog.mjs`.
  Don't edit `catalog-data.js` directly.
- To rename more files: add to `rename-map.mjs`, then run **freeze-classification.mjs
  → rename-files.mjs → build-catalog.mjs** in that order. Renaming is reversible via
  `rename-files.mjs --undo` (uses `rename-log.csv`).
- **Use the brand tokens, never hard-code** (per the kit's AGENTS.md): flat colour,
  no gradients/glow, crisp radii, cool petrol-tinted shadows, accent-bar elevation,
  sentence-case headlines, no emoji/exclamation marks.
- Catalogue links point at exact filenames; keep names and `classification.json` in sync.

## Provenance — MARVEL Learnings
The learnings were extracted by text-mining the Word deliverables (unzip
`word/document.xml`, strip tags) and reading the key passages — country reports,
roadmaps, the IFM paper, workshop reports, the EWS mapping and the assessment tool.
Each card cites the specific source(s). PDFs were not text-mined (no `pdftotext`);
to broaden coverage, read key PDFs with the Read tool's `pages` param and add items
to the `L` array. The extraction was done in the scratchpad (not committed).

- The kit's logo/mosaics were extracted from raster PNGs; request the original
  `.ai`/SVG (and the standard blue FAO lockup) for production-grade output.
- Whether FireHub grows from a catalogue into a full synthesised knowledge site
  (hub + outcome/country/course pages, i18n) — DRIP is the template if so.
- True duplicate files were renamed with a ` (copy)` suffix; the catalogue flags them
  as "possible duplicate" but does not delete them — review and remove if confirmed.
